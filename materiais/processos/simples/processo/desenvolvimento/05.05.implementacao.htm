Detalhamento das atividades

05 - Desenvolvimento

05.05. Implementar solução
Finalidade
Esta atividade descreve como deverá ser realizado as atividades de implementação ou desenvolvimento do produto de software.
Observações
Esta atividade da fase de desenvolvimento contém três sub-atividades:
  • Implementar Banco de Dados;
  • Implementar Código;
  • Implementar Interface Visual.
Estas três sub-atividades ocorrem paralelamente entre si, sendo necessário comunicação entre os recursos envolvidos para alcance eficaz dos objetivos desta atividade.


05.05.01. Implementar Banco de Dados.
Artefatos de Entrada
DCA - Documento de Configuração e Ambiente
DAQ - Documento de Arquitetura
DAN - Documentos de Análise
DER - Diagrama Entidade-Relacionamento
Artefatos de Saída
COM - Componente do Produto
Papel Responsável:
DES -Desenvolvedor
Papel Participante:
ANL - Analista de Sistemas
Ferramentas
Ferramenta de Desenvolvimento de Software
Ferramenta SGDB
Microsoft Visio
Passos
  1. Analisar requisitos do produto,
  2. Analisar Diagrama Entidade-Relacionamento Conceitual;
  3. Realizar projeto físico do Diagrama Entidade-Relacionamento Conceitual;
  4. Desenvolver e/ou revisar entidades e estabelecer e/ou revisar relacionamentos conforme o especificado;
  5. Realizar teste da base.
Passos Alternativos
  1. No passo 2, caso a base de dados não tenha sofrido modificações, terminar esta sub-atividade específica.
  2. No passo 3, caso a especificação não tenha sido aceita na atividade de implementação da solução, solicitar ao Analista de Sistemas responsável uma revisão da especificação. Continuar na atividade Revisar e/ou elaborar especificação e interface visual.
  3. No passo 4, caso o teste da base não tenha tido êxito, reiniciar no passo 3.
Observações

Nenhuma

DES -Desenvolvedor
Papel Participante:
ANL - Analista de Sistemas
Ferramentas
Ferramenta de Desenvolvimento de Software
Enterprise Architech
Microsoft Word
Passos
  1. Analisar requisitos do produto,
  2. Analisar arquitetura do projeto, configuração e ambiente;
  3. Realizar implementação de código.
    Para projetos WEB: projetar a especificação de casos de uso; Desenvolver casos de uso baseados no projeto da especificação;
    Para projetos de sistemas legados: desenvolver código segundo o especificado no Documento de Descrição Técnica;
  4. Realizar teste de desenvolvedor.

 
Passos Alternativos
  1. No passo 2, caso a arquitetura e a configuração e ambiente não tenham sido modificados, continuar no passo 3.
  2. No passo 3, caso a Documentação de Análise desenvolvida não tenha sido aceita na atividade de implementação da solução, solicitar ao Analista de Sistemas responsável uma revisão da Documentação de Análise e da especificação. Continuar na atividade Revisar e/ou elaborar especificação e interface visual.
  3. No passo 4, caso o teste não tenha sido bem sucedido, reiniciar no passo 3;
