05 - Desenvolvimento

05.01. Projetar ou refinar arquitetura
Finalidade
  • Definir uma sugestão de arquitetura para o sistema com base na experiência obtida com sistemas ou domínios de problema semelhantes;
  • Definir os padrões de arquitetura, os principais mecanismos e as convenções de modelagem para o sistema;
  • Definir a estratégia de reutilização.

Definição do RUP
Observações

Esta atividade seguirá o modelo proposto para os estilos:

  • Arquitetural de chamada e retorno, para sistemas baseados em paradigmas de desenvolvimento estruturados;
  •  Arquitetural em camadas, para sistemas baseados em paradigmas de desenvolvimento Orientados à Objetos.

Passos
  1. Fazer análise arquitetural dos requisitos;
  2. Desenvolver e/ou refinar arquitetura;
  3. Desenvolver e/ou revisar Documento de Arquitetura de Software.
Passos Alternativos
  1. No passo 2, caso a arquitetura seja modificada ou seja nova, demonstrar, descrever e validar a arquitetura em tempo de execução. Continuar no passo 3.
